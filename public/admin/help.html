Admin and editing help

The S3S site can be edited through Decap CMS or manually in GitHub/local files. GitHub controls who can publish changes.

Login and authorization

If admin login returns a server error, Cloudflare Pages probably needs GitHub OAuth variables.

Add these environment variables to the Cloudflare Pages project and redeploy:

  • GITHUB_CLIENT_ID
  • GITHUB_CLIENT_SECRET

The GitHub OAuth app callback URL should be:

https://website-bgu.pages.dev/api/callback

If GitHub says the RCIDS3S organization restricts OAuth apps, request organization access for RCID S3S CMS, then have an organization owner approve it under the OAuth app policy.

People and permissions

Editors need GitHub accounts with access to RCIDS3S/website. GitHub, not the CMS, grants and revokes that access. See the People and permissions screen for onboarding, offboarding, team roles, and direct links to the appropriate GitHub settings.

The practical handoff model is:

  • site-admins: repository Admin or Maintain access.
  • site-editors: repository Write access.
  • radar-maintainers: repository Write access plus code ownership for Opportunity Radar machinery.
  • site-viewers: repository Read access.

The CMS is configured with an editorial workflow, so content can move through draft, review, and publish states inside Decap.

Manual editing map

Most editors should start in the Decap admin. The sections are organized to match the site: homepage settings, events, resources, officers, the Opportunity Radar introduction and source suggestion queue, and the Legacy Archive.

Small label above title means the short all-caps style label that appears above a larger heading, such as “Clemson University / RCID” above the homepage headline.

Manual edits can still be made in GitHub or locally by changing these files:

  • Homepage hero: src/data/site/hero.json
  • Site title, subtitle, SEO: src/data/site/identity.json
  • Navigation labels: src/data/site/navigation.json
  • Student desk: src/data/site/student-desk.json and src/data/quick-links.json
  • Events, manual event cards, and calendar setup: src/data/events.json
  • Resources: src/data/resources.json
  • Officers: src/data/officers.json
  • About: src/data/about.json
  • Contact: src/data/site/contact.json
  • Footer: src/data/site/footer.json
  • Opportunity Radar introduction: src/data/opportunities-page.json
  • Opportunity Radar source suggestions: src/data/opportunity-source-suggestions.json
  • Legacy archive page copy and filter labels: src/data/archive/settings.json
  • Legacy archive posts: src/data/archive/posts.json
  • Images: public/assets/images/

Opportunity Radar boundary

Routine editors can change only the Radar's title and introductory copy, plus a safe queue of suggested source URLs. A suggested URL does not enter the live feed automatically. The published feed, generated candidates, source list, parsers, filter behavior, map behavior, labels, and automation are protected maintainer files.

Future maintainers should use the read-only Radar maintainer guide. It exposes the architecture, code map, refresh procedure, recovery steps, and takeover sequence without placing production controls in the CMS.

Manual GitHub workflow

  1. Open the GitHub repository.
  2. Open the file you need to edit.
  3. Use the pencil icon to edit.
  4. Keep JSON punctuation intact: quotation marks, commas, brackets, and braces matter.
  5. Create a branch and pull request. Opportunity Radar system changes require validation and code-owner review.
