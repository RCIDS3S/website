Opportunity Radar maintainer guide

This is the deliberate handoff surface for the Radar. It explains where the machinery lives without turning production code into ordinary CMS fields.

Routine editors: use the Opportunity Radar introduction editor and source suggestion queue only. Live source configuration, filters, parsers, automation, and generated items require a reviewed code change.

How the feed works

  1. Routine editors may suggest source URLs in a safe queue.
  2. Maintainers review suggestions and promote only suitable sources into the protected configuration.
  3. Configured sources define what the Radar monitors.
  4. A scheduled GitHub workflow runs the refresh script every day.
  5. The script fetches, parses, scores, filters, and deduplicates candidate opportunities.
  6. Automation writes a private audit set and the public feed data.
  7. The Astro page groups and renders the public feed using protected UI configuration.
  8. Cloudflare Pages publishes the validated result.

Safe takeover sequence

  1. Join the radar-maintainers GitHub team.
  2. Read the full runbook and trace the six-step flow above through the linked files.
  3. Run the site build without changing the feed.
  4. Run one manual refresh and inspect the generated diff.
  5. Make one small source change on a branch.
  6. Open a pull request, pass validation, and obtain code-owner review.

Emergency recovery

Revert the most recent Radar change, run the update workflow manually, confirm the validation workflow passes, and inspect the resulting Cloudflare deployment. If one source is causing the problem, move it to watch mode in a reviewed change before refreshing again.
